지난 달 테크 세미나 'Docker를 활용한 효율적 개발 및 보안전략'에서 제가 인상 깊게 들었던 부분에 대한 후기입니다.

가장 인상 깊었던 부분은 도커를 활용한 빠른 개발 속도를 다룬 부분에서 Docker init 명령어를 활용한 부분입니다. 조금 부끄럽지만 도커를 각 잡고 공부해본 적이 없고 사용할 때만 문서의 특정 부분만 참고하여 개발한 터라, 도커에 대한 지식이 거의 없습니다. 그런데 최근 업무로 파이썬 어플리케이션을 개발하면서 docker compose와 Dockerfile을 작성할 일이 생겼습니다. Docker init을 몰랐던 전 한글자 한글자 작성했었죠... 

Docker init 명령어는 자동으로 프로젝트의 언어를 감지하여 compose.yaml, Dockerfile, .dockerignore를 베스트 프랙티스를 상당 부분 적용하여 작성해줍니다. 

실제로 기존 프로젝트에 사용해보았는데요. 

compose.yaml -> db 관련 부분은 주석으로 가이드라인을 제공해주고 파이썬 관련 부분도 잘 작성해주었습니다.

Dockerfile -> 맨 처음 도커를 사용할 때 Dockerfile 부분이 좀 난감했는데 (옵션 등을 잘 몰라서) init을 활용하니 이 과정이 정말 간편해졌습니다. 

.dockerignore -> 이 부분도 자동으로 작성해주니 정말 시간 절약!

README.Docker.md -> 도커 실행 방법을 안내해주는 리드미도 작성해준답니다. 사실 문서 작업이 제일 귀찮잖아요 ㅎㅎ..

다른 흥미로운 주제들도 많이 다뤄주셨는데, 가장 감동 받은 부분 위주로 후기를 작성해보았습니다. 저는 프로젝트 관련 설정/셋업 부분이 전체 작업량에서 2~30%는 차지한다고 느꼈는데, 도커를 활용하면 이를 상당 부분 단축할 수 있을 것 같다고 느꼈습니다. 이전처럼 깔짝 대지 말고 문서를 좀 깊게 읽어봐야겠어요...
